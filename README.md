# Integração Supabase + Z-API

Script em Python que lê até três contatos do Supabase e envia, via Z-API:

> Olá, `<nome_contato>` tudo bem com você?

## 1. Criar a tabela no Supabase

Execute no **SQL Editor**:

```sql
create table contatos (
  id bigint generated by default as identity primary key,
  nome text not null,
  telefone text not null,
  created_at timestamptz not null default now()
);

insert into contatos (nome, telefone) values
  ('Mateus', '5511999999999'),
  ('João', '5511888888888'),
  ('Maria', '5511777777777');
```

O telefone deve conter código do país e DDD, somente com números.

## 2. Configurar o ambiente

Requer Python 3.10 ou superior.

```bash
python -m venv .venv
```

Ative o ambiente virtual:

```powershell
# Windows PowerShell
.\.venv\Scripts\Activate.ps1
```

```bash
# Linux/macOS
source .venv/bin/activate
```

Instale as dependências e crie o `.env`:

```bash
pip install -r requirements.txt
```

```powershell
# Windows PowerShell
Copy-Item .env.example .env
```

```bash
# Linux/macOS
cp .env.example .env
```

Preencha:

```dotenv
SUPABASE_URL=https://seu-projeto.supabase.co
SUPABASE_KEY=sua_chave_service_role
ZAPI_INSTANCE_ID=id_da_instancia
ZAPI_TOKEN=token_da_instancia
ZAPI_CLIENT_TOKEN=client_token
```

Use a chave `service_role` somente neste script executado no backend. Nunca envie
o `.env` ao GitHub. O `ZAPI_CLIENT_TOKEN` é necessário quando a segurança por
Client Token estiver habilitada na conta Z-API.

## 3. Executar

```bash
python main.py
```

O processo retorna código `0` quando todos os contatos válidos forem enviados e
`1` quando houver falha de configuração, banco ou envio.

## Testes

Os testes simulam Supabase e Z-API; nenhuma mensagem real é enviada:

